# Sanity Setup Guide for New Account

## Overview
This guide will help you set up a new Sanity account while keeping the exact same schema structure and UI.

## Step-by-Step Setup

### Step 1: Create New Sanity Project

1. Go to [sanity.io](https://www.sanity.io) and sign up/login
2. Create a new project
3. Note down your **Project ID** (you'll find it in project settings)
4. Choose a dataset name (default is usually "production")

### Step 2: Install Sanity CLI (if not already installed)

```bash
npm install -g @sanity/cli
```

### Step 3: Initialize Sanity in Your Project

Navigate to the `sanity_backend` directory:

```bash
cd sanity_backend
```

Initialize Sanity (this will create necessary config files):

```bash
sanity init --project <YOUR_NEW_PROJECT_ID> --dataset production
```

**OR** if you want to manually set it up (recommended to keep existing structure):

1. Update `sanity.config.js` with your new project ID
2. Update `sanity.cli.js` with your new project ID
3. Run `npm install` to install dependencies
4. Run `sanity login` to authenticate
5. Run `sanity deploy` to deploy your studio

### Step 4: Update Configuration Files

#### Update `sanity_backend/sanity.config.js`:
```javascript
projectId: 'YOUR_NEW_PROJECT_ID',  // Replace with your new project ID
dataset: 'production',  // Or your chosen dataset name
```

#### Update `sanity_backend/sanity.cli.js`:
```javascript
projectId: 'YOUR_NEW_PROJECT_ID',  // Replace with your new project ID
dataset: 'production',  // Or your chosen dataset name
```

#### Update `src/config/sanity.js`:
```javascript
projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID || "YOUR_NEW_PROJECT_ID",
dataset: process.env.NEXT_PUBLIC_SANITY_DATASET || "production",
```

#### Create/Update `.env.local` in root directory:
```env
NEXT_PUBLIC_SANITY_PROJECT_ID=YOUR_NEW_PROJECT_ID
NEXT_PUBLIC_SANITY_DATASET=production
NEXT_PUBLIC_SANITY_API_VERSION=2023-05-03
NEXT_PUBLIC_SANITY_USE_CDN=false
```

### Step 5: Deploy Schema to New Sanity Project

1. Make sure you're logged in:
```bash
cd sanity_backend
sanity login
```

2. Deploy your studio:
```bash
sanity deploy
```

This will deploy your Sanity Studio with all the schema definitions to your new project.

### Step 6: Verify Setup

1. Start the Sanity Studio locally:
```bash
cd sanity_backend
npm run dev
```

2. You should see your Sanity Studio with all content types (Post, Magazine, Category, etc.)

3. Test the frontend connection:
```bash
# From root directory
npm run dev
```

4. Verify that the frontend can connect to your new Sanity project (check browser console for any errors)

## Files Summary

### ✅ Copy These Files (Schema Structure):
- `sanity_backend/schemaTypes/` - **ALL FILES** (post.js, magazine.js, category.js, author.js, youtube.js, brandlogo.js, blockContent.js, index.js)
- `sanity_backend/package.json` - Dependencies

### 🔧 Update These Files (Project ID):
- `sanity_backend/sanity.config.js` - Update projectId
- `sanity_backend/sanity.cli.js` - Update projectId  
- `src/config/sanity.js` - Update projectId (or use env vars)
- `.env.local` - Create/update with new credentials

### ❌ Don't Copy (Auto-generated):
- `node_modules/` - Run `npm install` instead
- `package-lock.json` - Will be regenerated
- `.sanity/` - Auto-generated by Sanity CLI
- `.next/` - Next.js build output

## Important Notes

1. **Schema Files**: All schema files in `sanity_backend/schemaTypes/` define your content structure. These MUST be copied exactly as they are.

2. **Project ID**: The project ID is unique to each Sanity project. You MUST update it in:
   - `sanity.config.js`
   - `sanity.cli.js`
   - `src/config/sanity.js` (or use environment variables)
   - `.env.local` file

3. **Dataset**: Usually "production" but can be customized. Make sure it matches in all config files.

4. **Content**: Your new Sanity project will be empty. You'll need to:
   - Create categories first
   - Then create posts and magazines
   - Upload brand logos
   - Add authors
   - Add YouTube videos

5. **Environment Variables**: Using `.env.local` is recommended for security. The frontend config file has fallback values, but env vars take precedence.

## Troubleshooting

### Issue: "Project not found"
- Verify your project ID is correct
- Make sure you're logged in: `sanity login`
- Check that the project exists in your Sanity dashboard

### Issue: "Cannot connect to Sanity"
- Check `.env.local` file exists and has correct values
- Restart Next.js dev server after changing env vars
- Verify CORS settings in Sanity dashboard (should allow your domain)

### Issue: "Schema validation errors"
- Make sure all schema files are copied correctly
- Check that `schemaTypes/index.js` exports all schemas
- Verify no syntax errors in schema files
