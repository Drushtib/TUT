# Quick Setup Checklist

## тЬЕ Pre-Setup (Do This First)

- [ ] Create new Sanity project at [sanity.io](https://sanity.io)
- [ ] Copy your new **Project ID** (8 characters, e.g., `abc123xy`)
- [ ] Note your **Dataset name** (usually `production`)

---

## ЁЯУЛ Setup Steps (In Order)

### Step 1: Install & Login
```bash
npm install -g @sanity/cli
sanity login
```
- [ ] Sanity CLI installed
- [ ] Logged in successfully

### Step 2: Navigate to Backend
```bash
cd sanity_backend
```
- [ ] In `sanity_backend` directory

### Step 3: Initialize Sanity
```bash
sanity init
# OR manually update config files (see Step 4)
```
- [ ] Sanity initialized OR config files ready to update

### Step 4: Update Config Files
Update these 3 files with your NEW project ID:

**File 1: `sanity_backend/sanity.config.js`**
- [ ] Changed `projectId: 'lres172w'` тЖТ `projectId: 'YOUR_NEW_ID'`

**File 2: `sanity_backend/sanity.cli.js`**
- [ ] Changed `projectId: 'lres172w'` тЖТ `projectId: 'YOUR_NEW_ID'`

**File 3: `src/config/sanity.js`**
- [ ] Changed fallback project ID to your new ID

**File 4: `.env.local` (in root directory)**
- [ ] Created file with:
  ```
  NEXT_PUBLIC_SANITY_PROJECT_ID=YOUR_NEW_ID
  NEXT_PUBLIC_SANITY_DATASET=production
  NEXT_PUBLIC_SANITY_API_VERSION=2023-05-03
  NEXT_PUBLIC_SANITY_USE_CDN=false
  ```

### Step 5: Install Dependencies
```bash
npm install
```
- [ ] Dependencies installed (creates `node_modules/`)

### Step 6: Deploy Schema
```bash
sanity deploy
# OR for local testing:
npm run dev
```
- [ ] Schema deployed to new Sanity project
- [ ] Can access Sanity Studio (online or localhost:3333)
- [ ] See all content types (Post, Magazine, Category, etc.)

### Step 7: Test Frontend
```bash
cd ..
npm run dev
```
- [ ] Frontend runs without errors
- [ ] No Sanity connection errors in browser console
- [ ] Frontend connects to new Sanity project

### Step 8: Create Initial Content
- [ ] Created at least one Category
- [ ] Created test Post
- [ ] Verified content displays on frontend

## тЭУ Common Questions

**Q: When do I copy schema files?**  
A: You don't need to! They're already in your project in `sanity_backend/schemaTypes/`

**Q: When do I run `npm install`?**  
A: After updating config files, in Step 5. This generates `node_modules/`

**Q: When do I deploy?**  
A: After installing dependencies, in Step 6. This pushes your schema to Sanity.

**Q: What if I get "project not found"?**  
A: Check that project ID is correct in all 3 config files and `.env.local`
